Markdown Style Guide

The rules below are intended to act as a guideline for writing documentation for the Rhombus platform. This is a working document.

Headers and Hierarchy

  • Avoid using headers with a "index" of 3 or higher (i.e. ### header or h3).

  • Precedence between hierarchical relationships in sections is represented by the "index" of the header. For example, an h2 header is assumed to the parent of subsequent h3 headers.

Content

  • Sections should begin with a paragraph description of its subject matter.

  • Section content should be supplemented by asides and code examples.

  • Avoid Line length in excess of 80 characters.

Asides

Generic Aside

Asides are used to highlight information. They are written in standard HTML markup. An aside uses an h3 for the title, and a p tag for the content.

"Notice" asides are used to highlight edge cases or "gotchas" that may be common errors, or may not be self-evident.
